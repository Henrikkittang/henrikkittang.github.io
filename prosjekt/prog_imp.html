Linear search

There is nothing fancy about linear search. Given an array of data, you start at one end and traverse the whole list one by one. It is the simplest algorithms and requires no preperation of the data, like sorting. Therfore it is frequently used everywhere. It it still worth mentioning beause you will here alot about it. Often it is used as a anchorpoint when comparing algorithms. The algorithm can be implemented simply by having a for loop go over the array while checking if the data at each index matches the one you are looking for

When looking for a piece of data in an array, using linear search you will in the worst case have to traverse the whole array to find the data you are looking for. This means that linear search performs in linear time, or O(n) where n is the length of the array

Binary search

Binary search is an alternative to linear search when searching through a collection of data. Binary search is a lot faster than linear search, but it only works on sorted sets of data. Therefor it also becomes important to know about data structures and algorithms that may streamline the program for higher performance.

Binary search is all about splitting the amount of data in half for each iteration or “guess”. You would to begin with, need to know the range you are searching in and if you should go higher or lower for each iteration.

In a range from 0 to 100, you would first guess 50, which would eliminate half the options. If the number is higher, 75. If you end up with a decimal number, then ceil it to the closest integer. It is a fairly easy algorithm to implement, with some of minor improvement to made in order to optimize the algorithm.

Some technical details: In a range from 0 to 2^n, you would be able to “guess” every odd number in n attempts, and every even number in under n attempts. This does not include 0 and 2^n.

Binary search is frequently used in tree-like data structures and sorted sets of data. Is usually always preferred over linear search because of its higher performance. Even though it’s a relatively easy algorithm to implement, it still requires efficient sorting algorithms to be preferable over linear search. 

                                import math

                                def binary_search(upper_range, num):
                                    """Binary search assumes lower bound of 0"""
                                    attempts = 0
                                    cur_guess = upper_range // 2
                                    comp_num = cur_guess
                                    while cur_guess != num:
                                        attempts += 1

                                        comp_num = math.ceil(comp_num / 2)

                                        if cur_guess > num:
                                            cur_guess -= comp_num
                                        else:
                                            cur_guess += comp_num
                                    return attempts + 1


                                print(binary_search(1000, 100))
